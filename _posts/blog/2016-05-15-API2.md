## REST API syntax

REST APIs use the HTTP protocol, which is the protocol for web browsing. This makes things very convenient because you can just use a web browser to try API calls. A REST API mainly contains three parts: 

* Resource (noun)
* Method (verb)
* Parameters

**Resources** as nouns. They are the things you want to work with, such as a post, a user, a pet ID, or an account. 

**Methods** are verbs/actions you want to perform to your resources (i.e. nouns). 
The most common HTTP methods/verbs include:

* GET: retrieve a resource
* POST: create a new resource
* PUT: update or replace an existing resource
* DELETE: delete a resource 

**Parameters** specify the conditions you need to complete the action/method. For example, when you use Google to search for a keyword, you are providing the keyword as the parameter and using the GET method to retrieve the relevant resources from the web. GET is the most frequently used method because for the majority of the web users, they only retrieve information from the web rather than updating or deleting anything. Some parameters are mandatory, which means if you don't specify them, the API will NOT work.  Some are optional, which means these parameters can be omitted. Each API has its own rules. Refer to the specific API documentation for details. 

Another important factor to keep in mind is, to perform actions such as POST, PUT and DELETE, most likely you will need permission/authorization. This means you need an API key to do those actions. Even the GET method is not always publicly available in many REST APIs. You need to register with them to get the API key. An API key can be submitted as part of the parameters. 


The REST API syntax is roughly as follows. 

```
{Method} http(s)://{domain}/resources?para1=value1&para2=value2...$api_key={your_api_key}
```

in which

* Methods are GET, POST, PUT, DELETE
* `{domain}/resources` is the resource you are trying to retrieve, create, update, or delete.  
* `?` is an important symbol to indicate the end of the resource specification and the beginning of the parameters. 
* para and value pairs and API_key and value pair specify the parameters needed for this API call.  

For example, in the previous post, I mentioned this public Google API for retrieving book info based on ISBN number. You will see it contains the three main parts: method, resource, and parameter.  
   
```
GET https://www.googleapis.com/books/v1/volumes?q=isbn:0747532699
```

Note: In a web browser, the GET method is the default method, which means you do not need to include the Get method in a web browser.
